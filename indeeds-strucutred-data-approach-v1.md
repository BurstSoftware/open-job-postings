Types of structured data
Organisations use various types of data to inspect, analyse and gain insights into company operations. Data comes in a variety of formats, from raw data from databases to information about social media posts for a company. There are two main types of data, structured data and unstructured data. Structured data is set by a business and is in a specific structure or format before it's stored. In this article, we are going to define what structured data is, look at the differences between structured data vs unstructured data and discuss tools used to get the most out of structured data.

What is structured data?
If you're interested in learning about how to define and describe a website to make it easier for online search engines to read and understand it, you may want to know the answer to the question 'what is structured data.'. To provide search engines with a better understanding of a website, you can use specific terminology and vocabulary. This vocabulary is Schema.org and can change web content into usable code for search engines to process easily. Search engines can read this code and then display search results with rich, meaningful data.
You can easily integrate into existing websites, which improves online visibility. For example, if you have a website that includes lots of cooking recipes, using structured data can improve how your search results appear on search engines. It gives search engines a richer pool of data to use with the available online content from your website. We call these rich results rich snippets and can improve how an audience interacts with a website search result.

What is unstructured data?
Unstructured data is raw information that lacks any predefined data model or lacks any predefined organisation. Unstructured data can be text-heavy but may include data points such as dates, numbers or facts. There may be ambiguities and irregularities in this type of content which makes it difficult to parse and understand when compared to data stored in a structured database. No-SQL databases, audio files and video files are all examples of unstructured data. Technology has advanced so storing and processing unstructured data is easy to do.
This has encouraged the development of new tools that can store special formats of unstructured data. MongoDB and Apache Giraph are examples of these specialised formats and can store documents or more nuanced tasks like storing relationships between nodes in a network. Analysing unstructured data has become an important aspect of Big Data to monitor the large portions of stored data. Companies use unstructured pictures, videos and audio meaningfully thanks to the expansion of Big Data.
Semi-structured data
Semi-structured data is neither structured nor unstructured and doesn't conform to formal structures of data models used in relational databases or data tables. Semi-structured data contains tags that can separate semantic elements and create hierarchies for records within data sets. It's referred to as a self-describing structure because of this. JSON and SML are two examples of semi-structured data. Semi-structured data exists as a category between structured and unstructured data because it's much easier to parse and analyse. Big data solutions can read semi-structured data which makes it easier to analyse structured data when compared to unstructured data.

Types of structured data
Structured data comes in many different forms, but it always requires a specific code format. Structured data could be anything from books to digital products on an online store. You can add more details to snippets in search results using structured data. Unfortunately, search engines don't always include a rich snippet for a web page even if it includes structured data. This oversight means that you can always check if your page contains a rich snippet and if it doesn't you can add one yourself.
What is the purpose of structured data?
Structured data allows you to provide more context and content for search engines, essentially communicating with them. For example, if you have a website with a recipe you can let search engines know about the ingredients required or how long it's going to take to prepare the meal. Search engines can understand the information provided instantly and find the best way to show it in search results.
This, in turn, improves your visibility online and increases traffic to the web page. In essence, we can use structured data as a tool to better communicate with search engines. It provides search engines with a rich pool of information about a specific web page. From here, search engines can use the information provided to populate a data-rich, informative search result for users. Doing it right increases your potential for more traffic and gives you a wider-reaching audience as a result.

What is Schema.org?
The most popular search engines have developed Schema.org to make it easier to use structured data. Schema.org lets you find all of the structured data markups that the main search engines support. It's a database that houses coding that helps with creating rich snippets for SEO purposes.Schema.org can find the markup required for specific web pages. So if you have an online store that sells t-shirts, Schema.org has the markup required to show what t-shirt colours you have in stock via a snippet.
There are lots of templates available on Schema.org, so take the time to find ones that could improve your search engine visibility. Schema.org is a directory of different code formats that you can use to communicate with search engines. There are lots of different types of code that you can use for different social media platforms too, including Open Graph for Facebook or Twitter Cards.
What is JSON-LD?
JSON-LD is one of the most common markups found on Schema.org. It's a framework for writing coding, just like Microdata or RDFa. JSON-LD is the preferred markup for most people as it's more robust, less likely to cause issues with websites.
What is the importance of structured data for SEO?
Structured data is important when performing SEO because it better facilitates communication between search engines and your website. Search engines seek out information about web pages so that they can populate search results effectively. Structured data makes it easier for search engines to do this, so using structured data improves your rankings on search results.
Structured data is also a good way to alter or adjust your web page snippet and change how it appears on search results. You can tailor it to show more information or refine it to show specific, relevant information such as promotional sales. We include all of this in a bid to encourage more traffic, page rankings and sales for a company or website. As structured data becomes more well used, the information provided by search engines becomes more relevant and informed.

Examples of structured data
To develop your understanding of structured data it's helpful to view it in context. Below are a few examples of structured data. You can add the following to the snippets of your website:
Organisation
Organisation structured data is a piece of markup that businesses can use to show they are a legitimate company. You can place this type of markup anywhere on your website but it's best suited for the home page or the about page.
Local business
Local business structured data gives a specific geographical location that your business operates from. This can help with local marketing and improve users located near your business finding you. If you have a single location, this markup can be on the home page, about page or contact page of a website. For multiple locations, you can add unique local business markup for every individual location.
Product
Product structured data is useful for companies that sell physical products. Using this type of markup greatly improves your organisation's visual impact on search results. Users can view things like pricing, stock availability and even ratings of your products from the search results.
Job posting
Job posting structured data is a relatively new markup used by search engines and it has proven to be very useful for job board sites, recruitment companies and businesses looking to hire new staff. Implementing this markup provides your company with a lot more visibility when people are looking for jobs, which can dramatically increase your pool of job candidates.
